node-mongo-store-api

Node.js v18.12.1

URL and Endpoints

The server is already up in https://e-commerce-store-api.onrender.com/

The endpoints currently available are:

  • api/v1/auth to get user authentication and access into products.
  • api/v1/products to get a list of products.

How to use?

Get user authentication

  1. Go to the URL + api/v1/auth endpoint.
  2. Add /register route to register as a new user or add \login route to get user authentication and access in products endpoint.
For now you can get authentication only from an API testing as Postman.

Go to products endpoint

  1. Go to the URL + api/v1/products endpoint.
  2. You can add the nexts queries properties to get the query results: brand, category, sort, fields, numFilter, page, limit
  3. Add /productID to get a specific product.
  4. Add /brands to get a unique list of brands.
  5. Add /categories to get a unique list of categories.
  6. Add /productID/image to get an image object of a specific product.

  • How to do a query?

    You can use the query parameter ? to chain a query property in a link.

    Example: .../api/v1/products + ? +

  • How to chain more than one query?

    You can use the ampersand parameter & to chain multiple queries

    Example: .../api/v1/products + ? + + & + ----> .../api/v1/products?name=^a&brand=x

  • Properties description

    1. name: accepts a regex value, example to use it: ?name=^a
    2. brand: only accepts brands values, example to use it: ?brand=nike
    3. category: only accepts category values, example to use it: ?category=shoes
    4. description: accepts regex value to match description content, example to use it: ?description=jogger
    5. sort: Use it to sort the products according to a property, example to use it: ?sort=price
    6. fields: Use it to get the products according to properties selected, example to use it: ?fields=price,name
    7. numFilter: Use it to filter the products according to price or rating, example to use it: ?numFilter=price>100,rating=4,discount=15
    8. page: Use it to get a page according to the default limit of products, example to use it: ?page=2
    9. limit: Use it to set the limit of products that can be shown, example to use it: `?page=2`
